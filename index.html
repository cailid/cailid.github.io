CAILID

Curated Artificial Intelligence Labeled Image Database

CAILID is an experimental database of high-quality manually curated and labeled images, suitable for testing, training, and fine-tuning machine learning models.

Purpose

  • Adding a small selection of very high quality images with a distinctive label format to a dataset enables models to learn that label format. Image generation prompts can then use the same text style to precisely tailor the content they want to create.
  • Many types of content that image generation models learn about has a small potential corpus. One could, for example, go through and label the bulk of an artist's entire work, potentially greatly improving future queries related to that topic.
  • Manually curated data is not nearly as susceptible to the "80s"-problem, where if you want to generate data for a particular topic you often won't get an accurate representation of it; exemplified in the faux cultural memory of glowing neon colors that is strongly associated with the 80s in automatically tagged databases.

Features

  • Each image is paired with a manually crafted text string in a common style that describes the content in detail.
  • In addition to the content description, the images have various metadata tags that among other things describe the medium, style, artist, and time period of the work. Notably, photographs are paired with information about the camera, lens, and settings used when taking the picture. As style is represented separately from the content, it can be differentiated from actual depictions of the artists themselves.
  • CAILID always documents what license each image in the database is covered by. Combined with the extensive tagging system for topics like violence, nudity, and if public figures are depicted, one can get precise control over what data is used during training.
  • Each image in the database is at least 1024x1024 pixels large, and free of any noticeable compression artifacts if close to that.
  • The database does not contain the images themselves, only links to the actual files. These must be privately fetched through the URL where they were originally hosted.

Labeling philosophy

Image labels are designed to accurately pin down the image in the theoretical space of all images that the database may contain for all features that could be of interest.

The labels "man, smiling", for example, provide many bits of information compared to no label at all. Adding "happy" to that same image provides few bits in comparison, but perhaps enough to be included if it eliminates a sufficiently significant cluster of images where smiling men are smiling for other reasons than that they are happy.

In comparison, one would likely not add "eyes" as a tag to the previous image. The overwhelmingly vast majority of images in the "smiling man" category feature people that do have eyes, meaning it does not provide sufficient information.

Status

Currently the amount of content in the database is fairly limited, and sourced from two locations for different purposes:

  • Unsplash, to get a wide selection aesthetically pleasing photographs that cover most labeling examples.
  • Wikimedia Commons, to label all high-quality publicly available oil paintings by Peder Severin Krøyer (1851-1909), so as to verify potential improvements in image generation of a particular topic if CAILID is included in a training set.
